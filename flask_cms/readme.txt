[[ flask cms 구조 ]]
-----------------------------
static : 정적 데이터가 위치하는 폴더. css, js, image, 업로드된 파일등
         자동으로 url이 세팅이 됨
         http://ip:port/static/파일명
-----------------------------
templates : render_template() 함수의 1번인자 파일이 위치한 폴더. html 위치
-----------------------------
service : db, 라우팅등등 python 코드 위치한 곳
-----------------------------
run.py : 프로그램 시작점
-----------------------------

세션 : 서버측에 저정하는 사용자의 정보
       서버 메모리(구현간단, 소량의 사용자 적합) 혹은 데이터베이스에 저장 관리 가능
       => 사용자가 로그인 되어 있음을 세션에 저장해서 체크후 페이지 접근등등에 활용
       이런 경우 데이터의 성격에 따라서는 페이지 요청시 데이터를 보내지 않고
       세션을 접근해서 데이터를 획득하여 사용하는 경우도 있음

쿠키 : 클라이언트측에 저장하는 사용자의 정보
